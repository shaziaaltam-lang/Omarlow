# دليل المشروع التقني (README)

مرحباً بكم في مستودع النظام القانوني والإداري المتقدم. هذا المشروع مبني باستخدام إطار عمل Laravel مع الالتزام بأحدث معايير هندسة البرمجيات.

## 📋 متطلبات النظام
لضمان عمل النظام بكفاءة، يجب توفر المتطلبات التالية:
- PHP 8.2 أو أحدث.
- Composer لإدارة المكتبات.
- Docker و Docker Compose لتشغيل بيئة التطوير.
- Redis للتخزين المؤقت.
- قاعدة بيانات متوافقة (MySQL أو PostgreSQL).

## 🏗️ هيكلية المجلدات
يعتمد المشروع على بنية منظمة ومقسّمة كالتالي:
- `app/Actions`: تحتوي على منطق العمل الأساسي.
- `app/Repositories`: تنفيذ نمط (Repository Pattern) لفصل منطق قواعد البيانات.
- `app/Services`: طبقة الخدمات المركزية لمعالجة العمليات المعقدة.
- `app/Http/Controllers`: المتحكمات للتعامل مع طلبات API والويب.
- `docs/`: التوثيق الفني الكامل للمشروع.

## 🛠️ نمط التطوير المتبع (Repository Pattern)
يستخدم هذا المشروع نمط المستودعات (Repository Pattern) لضمان:
1. **فصل المهام (Separation of Concerns)**: حيث يتم عزل كود الاستعلام عن قاعدة البيانات عن طبقة المنطق.
2. **سهولة الاختبار**: يمكننا بسهولة عمل Mock للمستودعات أثناء كتابة الاختبارات الوحدوية.
3. **مرونة التغيير**: إمكانية تغيير مصدر البيانات دون التأثير على المتحكمات.

## 🚀 كيفية التشغيل
1. نسخ ملف البيئة: `cp .env.example .env`.
2. تثبيت المكتبات: `composer install`.
3. تشغيل الحاويات: `docker-compose up -d`.
4. تنفيذ الهجرات: `php artisan migrate --seed`.

## 🧪 الاختبارات
لضمان استقرار النظام، يرجى تشغيل مجموعة الاختبارات قبل رفع أي تعديلات:
- لاختبار الميزات: `php artisan test --testsuite=Feature`.
- لاختبار الوحدات: `php artisan test --testsuite=Unit`.

## ⚙️ قائمة الأوامر المخصصة
- `php artisan app:cleanup`: تنظيف قاعدة البيانات والملفات المؤقتة.
- `php artisan system:check`: فحص صحة النظام والخدمات المرتبطة.
