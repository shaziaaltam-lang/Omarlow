# نظام إدارة القضايا القانونية الرقمي

## نبذة عن المشروع
هذا المشروع عبارة عن منصة برمجية متطورة مصممة لإدارة القضايا القانونية، العملاء، والفواتير بكفاءة عالية. يعتمد النظام على تقنيات Laravel الحديثة مع بنية تحتية قوية تدعم التوسع والأمان.

## المميزات التقنية
- **البنية البرمجية**: نمط (Repository Pattern) لفصل منطق الأعمال عن الوصول للبيانات.
- **الأمان**: حماية متكاملة عبر الـ Middleware، دعم التحقق بخطوتين (2FA)، وتطبيق سياسات الوصول (Policies).
- **الأداء**: استخدام Redis للتخزين المؤقت (Caching) و Elasticsearch للبحث المتقدم.
- **الذكاء الاصطناعي**: دمج خدمات OCR لمعالجة المستندات القانونية.
- **الواجهات**: توثيق كامل للـ API باستخدام معايير OpenAPI/Swagger.

## متطلبات التشغيل
- PHP 8.2+
- Composer 2.x
- Node.js & NPM
- Docker & Docker Compose (لبيئة التطوير والإنتاج)
- Redis

## خطوات التثبيت
1. استنساخ المستودع: `git clone <repository_url>`
2. تثبيت الحزم: `composer install && npm install`
3. إعداد ملف البيئة: `cp .env.example .env`
4. تشغيل الحاويات: `docker-compose up -d`
5. تهجير قاعدة البيانات: `php artisan migrate --seed`
6. توليد مفتاح التطبيق: `php artisan key:generate`

## أوامر Artisan الأساسية
- `php artisan db:seed`: لتعبئة البيانات الأولية.
- `php artisan queue:work`: لتشغيل طابور المعالجة الخلفية.
- `php artisan system:health-check`: للتأكد من سلامة النظام.
- `php artisan cleanup:database`: لتنظيف سجلات البيانات القديمة.

## هيكلية المشروع
- `app/`: يحتوي على الكود المصدري ومنطق الأعمال.
- `database/`: يحتوي على هجرات الجداول (Migrations) والمصنفات (Seeders).
- `routes/`: تعريف مسارات الويب والـ API.
- `tests/`: اختبارات الوحدة واختبارات الميزات (Feature Tests).
