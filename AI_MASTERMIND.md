# 🧠 العقل المدبر للذكاء الاصطناعي | AI Mastermind Reference

هذا المستند يمثل مرجعاً تقنياً مستمراً وعقلاً مدبراً لتوثيق وتوجيه عمليات التطوير الفنية في هذا المستودع. يقوم الذكاء الاصطناعي بقراءته والرجوع إليه في كل عملية (إنشاء، تعديل، أو حذف) لضمان اتساق الكود وجودته.

## 📌 1. الرؤية وأهداف المشروع الحالية (Project Vision & Goals)
- بناء وتحديث مكونات برمجية متناسقة للغاية وذات واجهة مستخدم مذهلة وسلسة.
- تطبيق أفضل معايير الأمان وحماية البيانات وتنبيه الحصص.
- توفير أفضل أداء تشغيلي للملفات ومعالجة متفوقة للأخطاء البرمجية.

## 🛠️ 2. القواعد البرمجية الصارمة (Strict Programming Rules)
1. **أكواد كاملة 100%**: منع الهلوسة البرمجية أو إظهار تعليقات تختصر الكود مثل "// rest of code" أو "// اكتب هنا باقي المكون". يجب توليد وتحديث الملف كاملاً بنسبة 100% دوماً.
2. **لغة المخرجات والتعليقات**: لغة التطبيق الأساسية ومقترحات التحسين والتعديل وسرد التغييرات تتم حصراً باللغة العربية الفصحى البليغة والراقية.
3. **توثيق الأكواد ومراجعتها**: كتابة تعليقات مفيدة ومختصرة باللغة العربية لشرح الأكواد المعقدة.
4. **تجنب الأعطال والتعديلات التخريبية**: عند تحديث الكود، يجب الحفاظ على سلامة الـ imports والمكونات المحيطة به دون التسبب في مسح كود أو تحطيم بنية التطبيق الحالية.

## 📝 3. سجل المهام والتعديلات المستمرة (Active Roadmap & Task Backlog)
- [ ] **تحليل بنية المستودع الأساسي**: تحديد الأماكن التي تحتاج لمرونة إضافية.
- [ ] **تحسين كفاءة المكونات الحالية**: تسريع الاستجابة وتبسيط كتابة المكونات.
- [ ] **بناء المكونات الناقصة**: سد أي فجوات هيكلية أو فنية يحددها الفحص الشامل للمستودع.
